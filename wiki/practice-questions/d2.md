# Practice Questions — Domain 2: Fundamentals of Generative AI

### Q1. In a generative AI text pipeline, what is a "token"?
- A. A security credential used to authenticate calls to Amazon Bedrock
- B. A basic unit of text (a word or sub-word) that the model processes
- C. A numeric vector representing the meaning of a document
- D. A single training epoch of a foundation model

<details><summary>Answer & explanation</summary>

**Answer: B**

A token is a basic unit of text (a word or part of a word) that the LLM processes; billing for many GenAI services is based on the number of input/output tokens. A meaning vector is an embedding (C), not a token.
</details>

### Q2. A team splits long documents into smaller passages before creating embeddings for a search application. What is this technique called?
- A. Tokenization
- B. Fine-tuning
- C. Chunking
- D. Quantization

<details><summary>Answer & explanation</summary>

**Answer: C**

Chunking is splitting long documents into smaller passages to embed and store in a vector store, enabling more precise retrieval (commonly used in RAG).
</details>

### Q3. Which statement best describes an embedding?
- A. A dense numeric vector that captures the semantic meaning of text or data
- B. A prompt template used to guide model output
- C. A managed endpoint that hosts a foundation model
- D. A compression format that reduces model file size

<details><summary>Answer & explanation</summary>

**Answer: A**

Embeddings are numeric vectors that represent semantic meaning; items with similar meaning are close together in the vector space, enabling similarity search.
</details>

### Q4. Which type of foundation model is specifically designed to generate high-quality images from text prompts?
- A. Transformer-based LLM
- B. Diffusion model
- C. Regression model
- D. Decision tree ensemble

<details><summary>Answer & explanation</summary>

**Answer: B**

Diffusion models (e.g. Stable Diffusion) generate images by gradually denoising from noise, a common architecture for image generation. LLMs are mainly for text.
</details>

### Q5. A company wants a single model that can accept both text and images as input and reason over them together. Which characteristic should the model have?
- A. Multi-modal
- B. Nondeterministic
- C. Provisioned throughput
- D. Zero-shot

<details><summary>Answer & explanation</summary>

**Answer: A**

Multi-modal models can accept and process multiple data types (text, image, audio…) at once. Zero-shot is a prompting technique, not a multi-modal capability.
</details>

### Q6. Which of the following are appropriate use cases for generative AI? (Choose TWO.)
- A. Summarizing long customer support transcripts
- B. Guaranteeing exact, repeatable numeric calculations for accounting ledgers
- C. Generating draft marketing copy and product descriptions
- D. Enforcing strict rule-based access control decisions
- E. Storing transactional records with ACID guarantees

<details><summary>Answer & explanation</summary>

**Answer: A, C**

GenAI is strong at summarization and creative content generation. Tasks that need absolutely exact results/hard rule-based decisions (B, D) or transactional storage (E) are not suitable due to its nondeterministic nature.
</details>

### Q7. Which sequence correctly represents stages of the foundation model lifecycle?
- A. Deployment → Data selection → Pre-training → Feedback
- B. Data selection → Model selection → Pre-training → Fine-tuning → Evaluation → Deployment → Feedback
- C. Fine-tuning → Pre-training → Data selection → Deployment
- D. Evaluation → Deployment → Data selection → Pre-training

<details><summary>Answer & explanation</summary>

**Answer: B**

The FM lifecycle goes from data selection, model selection, pre-training, fine-tuning, evaluation, deployment, then collecting feedback for improvement — the correct logical order.
</details>

### Q8. A generative AI chatbot occasionally produces confident but factually incorrect answers. What is this limitation called?
- A. Overfitting
- B. Hallucination
- C. Data drift
- D. Underfitting

<details><summary>Answer & explanation</summary>

**Answer: B**

Hallucination is when the model produces content that sounds plausible but is factually wrong — a characteristic limitation of GenAI that requires verification (e.g. using RAG, human review).
</details>

### Q9. Giving the same prompt to a generative AI model can return different outputs on different runs. Which limitation does this illustrate?
- A. Interpretability
- B. Nondeterminism
- C. Latency
- D. Bias

<details><summary>Answer & explanation</summary>

**Answer: B**

Nondeterminism means the same input can produce different outputs, a limitation to note when you need consistent, reproducible results.
</details>

### Q10. An online retailer deploys a generative AI shopping assistant. Which metric BEST measures whether the assistant increases the percentage of visitors who complete a purchase?

### Q14. A company runs a high-volume production chatbot on Amazon Bedrock and needs guaranteed, consistent throughput with predictable latency. Which pricing/deployment option is MOST appropriate, and what is the tradeoff?
- A. On-demand token-based pricing; lowest possible baseline cost with no capacity commitment
- B. Provisioned Throughput; guaranteed capacity for steady high volume, but you commit to (and pay for) reserved capacity
- C. PartyRock free playground; production-grade SLAs at no cost
- D. Amazon Q Business; unlimited image generation at a flat rate

<details><summary>Answer & explanation</summary>

**Answer: B**

Provisioned Throughput on Bedrock guarantees stable capacity/throughput for high load and predictable latency, in exchange for committing to pay for the reserved capacity. Token-based (on-demand) is more flexible but does not guarantee throughput; PartyRock is only an experimentation playground, not for production.
</details>

### Q15. Which characteristic of generative AI is an ADVANTAGE that allows one foundation model to handle many different tasks (summarizing, drafting, translating) without building a separate model for each?
- A. Determinism
- B. Adaptability
- C. Full interpretability
- D. Zero latency

<details><summary>Answer & explanation</summary>

**Answer: B**

Adaptability is a core advantage of GenAI: one FM can adapt to many tasks via prompting/customization without a separate model for each. Determinism and full interpretability are precisely what GenAI usually LACKS.
</details>

### Q22. During the foundation model lifecycle, at which stage is a large model FIRST trained on a broad, unlabeled corpus to learn general language patterns BEFORE any task-specific adaptation?
- A. Fine-tuning
- B. Pre-training
- C. Deployment
- D. Feedback

<details><summary>Answer & explanation</summary>

**Answer: B**

Pre-training is the stage where the model learns general language patterns from a large (usually unlabeled) dataset. Fine-tuning happens AFTER, to adapt to a specific task.
</details>

### Q23. In the foundation model lifecycle, which stage focuses on collecting real-world user responses after deployment to guide future improvements or retraining?
- A. Data selection
- B. Pre-training
- C. Feedback
- D. Model selection

<details><summary>Answer & explanation</summary>

**Answer: C**

Feedback is the stage of collecting user responses/signals after deployment to improve or retrain the model, closing the FM lifecycle loop.
</details>

### Q24. How does a diffusion model fundamentally generate an image?
- A. By retrieving the closest matching image from a training database
- B. By iteratively removing noise from a random starting point until a coherent image forms
- C. By tokenizing pixels and predicting the next word
- D. By applying fixed if-else rules to color values

<details><summary>Answer & explanation</summary>

**Answer: B**

Diffusion models generate images by gradually denoising from random noise until a coherent image forms. They do not look up existing images or use hard rules.
</details>

### Q28. Which are advantages of using managed AWS generative AI services (such as Amazon Bedrock) instead of self-hosting foundation models? (Choose TWO.)
- A. Lower barrier to entry and faster speed to market
- B. Eliminates the need to consider security or compliance entirely

### Q29. A startup expects highly variable, unpredictable traffic to its new generative AI feature and wants to pay only for what it uses while validating demand. On Amazon Bedrock, which cost approach best matches, and why?
- A. Provisioned Throughput, because committing to reserved capacity is always cheapest
- B. On-demand token-based pricing, because it scales with actual usage with no capacity commitment
- C. A custom model, because training your own is cheaper than any inference option
- D. PartyRock paid tier, because it offers production SLAs

<details><summary>Answer & explanation</summary>

**Answer: B**

For variable, hard-to-predict traffic where you want to pay per use, on-demand token-based pricing fits because it scales with actual usage with no capacity commitment. Provisioned Throughput is only economical for high, steady load.
</details>

### Q30. An enterprise is deciding between using a foundation model as-is via prompting versus building a custom model. Which statement about the cost tradeoff is MOST accurate?
- A. Custom models always cost less because you avoid all inference charges
- B. Custom (or fine-tuned) models can improve task fit but add data preparation, training, and maintenance costs compared with using a base model via prompting
- C. Using a base model via prompting requires more upfront training cost than a custom model
- D. Both options have identical cost and effort

<details><summary>Answer & explanation</summary>

**Answer: B**

Custom/fine-tuned models can improve task fit but incur additional data-preparation, training, and maintenance costs compared with just using a base model via prompting. This is a key cost consideration when choosing a customization approach.
</details>
